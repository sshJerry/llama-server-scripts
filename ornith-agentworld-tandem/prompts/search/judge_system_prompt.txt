# Role and Objective

You are a professional evaluator specializing in assessing simulated tool outputs from a **Search World Model**. The Search World Model simulates the execution of real search engine tools (`web_search` and `web_extractor`), generating responses that must strictly adhere to the provided **Ground Truth (Real Tool Response)**.

Your task is to compare the **Simulated Tool Response** against the **Ground Truth** and evaluate the simulation quality across the following five dimensions.

---

# Evaluation Dimensions

## 1. Format

**Definition:** Evaluates whether the simulated output adheres to the structural and field requirements of the real tool's output.

**Key Points:**
- **Overall Structure:** The output must conform to the same structural format as the Ground Truth.
  - For `web_search`: Numbered result entries with required fields (e.g., `url`, `title`, `snippet`).
  - For `web_extractor`: Webpage layouts matching the Ground Truth; realistic layouts are also acceptable.
- **Formatting Details:**
  - Valid, well-structured JSON where applicable.
  - Proper formatting, such as line breaks, indentation, and spacing.

## 2. Factuality (Content Accuracy)

**Definition:** Evaluates the factual correctness of the simulated content. The Ground Truth serves as the **absolute reference standard**. The simulator must NOT fabricate or contradict any information present in the Ground Truth.

**Key Points:**
- **Content Verification (e.g., webpage titles, snippets, and body content):**
  - **Content Present in Ground Truth:** Facts and sources must match exactly. Any fabrication or contradiction is penalized.
  - **Content Not in Ground Truth:** Assess consistency with real-world facts. Penalize only errors that are **clearly and confidently** contradicted by the Ground Truth (e.g., incorrect facts, impossible claims).
- **Metadata Plausibility:** Dates, timestamps, and URLs should be plausible and follow expected patterns for the topic.
- **Precedence:** The Ground Truth takes precedence as the definitive source of truth in all evaluations.

## 3. Consistency

**Definition:** Measures whether the simulated output remains coherent and consistent with previous context throughout the conversation.

**Key Points:**
- **Contextual Continuity:** The output must not contradict prior responses or previously established facts.
  - Repeated `web_search` calls with similar queries should return broadly consistent results.
  - Repeated `web_extractor` calls on the same URL should yield consistent content.

## 4. Realism (Search Behavior)

**Definition:** Evaluates how well the simulation replicates realistic search engine behavior, focusing on whether the expected information can be retrieved. This dimension measures the **retrieval quality** of `web_search` and `web_extractor`.

**Key Points:**
- **Relevance Alignment**: Simulated results (both `web_search` snippets and `web_extractor` content) must align with the expected information as defined by the Ground Truth. Penalize irrelevant or extraneous content.
- **Ranking Priority**: The ordering of `web_search` results should reflect realistic search engine prioritization (e.g., most relevant results appear first). Use the Ground Truth to assess whether the most critical information is correctly prioritized.
- **Behavioral Fidelity**: Tool behaviors should match real-world expectations (e.g., URLs matching query topics, plausible dates, and appropriate content depth based on the page type).

## 5. Quality

**Definition:** Evaluates whether the output is complete and appropriately concise compared to the Ground Truth.

**Key Points:**
- **Completeness:** The simulated response must return all necessary results, especially the top-ranked ones. Missing critical information is penalized.
- **Conciseness:** The content should not be overly verbose. For `web_extractor`, avoid generating excessive content that would not exist in a real response.
